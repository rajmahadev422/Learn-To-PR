## 👋 Welcome!

If you've never contributed to open source before, **you're in the right place.**  
This repo exists for one reason: to give you a safe, welcoming space to make your first GitHub contribution.

No complex code. No scary reviews. Just you, learning how open source works — one issue and PR at a time.

---

## 🚀 What You Can Do Here

| Action | What It Means |
| --- | --- |
| 🐛 **Open an Issue** | Report a bug, suggest an idea, or ask a question |
| 🔀 **Open a Pull Request** | Submit a change to the repository |
| 💬 **Comment on Issues** | Discuss ideas with the community |
| ✅ **Review a PR** | Give feedback on someone else's contribution |

---

## 📋 How to Make Your First Issue

1. Go to the **[Issues](../../issues)** tab
2. Click **"New Issue"**
3. Choose a title (e.g. `Add: My name to contributors list`)
4. Describe what you want to do or report
5. Click **"Submit new issue"** 🎉

> 💡 **Tip:** Not sure what to write? Try introducing yourself in `docs/others/[your-github-username].md`: *"Hi! I'm [name] and I want to make my first contribution."*. And also add your **github-user-name** in `contributors.md`.

| Label | Meaning |
| --- | --- |
| `good first issue` | Perfect for beginners |
| `help wanted` | We'd love your contribution |
| `bug` | Something isn't working |
| `enhancement` | A new feature or improvement |
| `question` | Asking for help or clarification |
| `documentation` | Improving or fixing docs |

## 🤝 Contribution Guidelines

- **Be kind.** Everyone here is learning. Treat others with respect.
- **Be patient.** Maintainers will review your PR as soon as possible.
- **Be descriptive.** Write clear titles and descriptions for issues and PRs.
- **One change per PR.** Keep pull requests small and focused.
- **Ask questions freely.** There are no silly questions here!
